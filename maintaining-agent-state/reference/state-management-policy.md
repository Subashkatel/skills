# State Management Policy

Active state is current truth, not a diary. Keep it short, reconcile it with the filesystem, and update it when reality changes.

Use structured JSON for status, test gates, evidence, and todo state. Use Markdown for explanations, deviations, and handoff notes. Use git notes for commit-bound rationale. Use lesson files only for reusable facts that future runs should remember.

Never trust memory over the repository, test output, profiler output, simulator results, or source documents.
